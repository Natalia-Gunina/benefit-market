## 1. Общие сведения

**Название системы:** Benefit Market

**Назначение:** Мультитенантная платформа управления корпоративными льготами с маркетплейсом провайдеров. Система обеспечивает полный цикл работы с программой льгот — от начисления баллов сотрудникам до оформления и отслеживания заказов.

**Тип приложения:** Одностраничное веб-приложение (SPA) с серверным рендерингом (SSR)

**Версия:** 0.1.0

**Целевые роли пользователей:**

| Роль | Описание |
|------|----------|
| `employee` | Сотрудник — просмотр каталога льгот, оформление заказов, управление балансом |
| `hr` | HR-администратор — управление политиками, сотрудниками, аналитика |
| `provider` | Провайдер — размещение предложений на маркетплейсе, управление заказами |
| `admin` | Системный администратор — полный контроль над платформой |

| Паттерн | Область применения |

## 5. Система аутентификации и авторизации

### 5.1. Аутентификация

Аутентификация реализована на базе **Supabase Auth** с использованием JWT-токенов.

**Процесс регистрации:**
1. Пользователь заполняет форму регистрации
2. Supabase Auth создаёт запись в `auth.users`
3. PostgreSQL-триггер (`00005_auth_trigger.sql`) автоматически создаёт запись в таблице `users` и инициализирует кошелёк (`wallets`)
4. Роль сохраняется в `user_metadata` и `app_metadata`

**Клиенты Supabase:**

| Клиент | Файл | Назначение |
|--------|------|-----------|
| Браузерный | `lib/supabase/client.ts` | Операции от имени текущего пользователя |
| Серверный | `lib/supabase/server.ts` | SSR-рендеринг с учётом аутентификации |
| Административный | `lib/supabase/admin.ts` | Операции с `service_role_key` (обход RLS) |

### 5.2. Авторизация (RBAC)

Ролевая модель доступа реализована на двух уровнях:

**Уровень приложения (Middleware):**
- `src/middleware.ts` перехватывает все запросы
- Извлекает роль пользователя из JWT-токена
- Перенаправляет на соответствующий дашборд
- Блокирует доступ к неразрешённым маршрутам

**Уровень базы данных (RLS):**
- **37 политик** обеспечивают изоляцию данных
- Все запросы фильтруются по `tenant_id`
- Сотрудники видят только свои записи
- HR-менеджеры — данные своего тенанта
- Администраторы — полный доступ

### 5.3. Демо-режим

При `NEXT_PUBLIC_DEMO_MODE=true`:
- Middleware пропускает проверку аутентификации
- Загружаются моковые данные из `lib/demo-data.ts`
- Позволяет демонстрировать систему без настройки Supabase

## 9. Развёртывание и CI/CD

### 9.1. Контейнеризация (Docker)

Используется **многоступенчатая сборка** на базе `node:20-alpine`:

| Этап | Назначение | Результат |
|------|-----------|----------|
| `base` | Базовый образ | Node.js 20 Alpine |
| `deps` | Установка зависимостей | `npm ci` (только production) |
| `builder` | Сборка приложения | `next build` (standalone output) |
| `runner` | Продуктивная среда | Минимальный образ с `node server.js` |

**Характеристики контейнера:**
- Порт: 3000
- Пользователь: `nextjs` (UID 1001, непривилегированный)
- Переменная: `NODE_ENV=production`
- Телеметрия: отключена (`NEXT_TELEMETRY_DISABLED=1`)

### 9.2. Docker Compose

| Файл | Назначение |
|------|-----------|
| `docker-compose.yml` | Локальная разработка (демо-режим) |
| `docker-compose.prod.yml` | Продуктивная среда (приложение + Caddy) |

**Продуктивная конфигурация:**
- **Сервис `app`**: образ из `ghcr.io`, healthcheck
- **Сервис `caddy`**: обратный прокси, SSL-терминация, порты 80/8443

### 9.3. CI/CD (GitHub Actions)

#### Пайплайн непрерывной интеграции (`ci.yml`)

## 10. Безопасность

### 10.1. Защита на уровне приложения

| Механизм | Реализация |
|----------|-----------|
| **Аутентификация** | JWT-токены через Supabase Auth |

| **Валидация входных данных** | Zod-схемы на всех API-эндпоинтах |
| **Санитизация** | Очистка пользовательского ввода от XSS (`lib/api/sanitize.ts`) |
| **Rate limiting** | Ограничение частоты запросов (`lib/api/rate-limiter.ts`) |

Настроены в `next.config.ts`:

| Заголовок | Значение |

| `X-Content-Type-Options` | `nosniff` |
| `Content-Security-Policy` | Ограничение источников скриптов, стилей, подключений |

### 10.3. Безопасность контейнера

- Непривилегированный пользователь `nextjs` (UID 1001)
- Минимальный Alpine-образ (уменьшенная поверхность атаки)
